Commenting out query



For a successful injection, we should close the open delimiter wrapped around the variable in the query, thereby making it possible for us to escape the string/integer boundary and execute SQL statements and also be able to handle the closing delimiter around the variable.

This can be done in two ways.
1. One way is by commenting out the rest of query.
2. Adding an extra delimiter with extra values to make the query syntactically correct.

a) On lesson 1.
We can use SQL comments to fix the syntax. MySQL uses three types of comments: --+, # , /* */, so our injection can be 1′ --+ or 1′ #.

URL: http://localhost/Less-1/?id=1'--+



QUERY: SELECT * FROM TABLE_NAME WHERE ID=’ 1′--+ ‘ LIMIT 0,1

b) On lesson 2.

We do not need any extra quotes to escape and inject queries as there are no quotes in the first place, therefore a simple commenting out should be fine.

URL: http://localhost/Less-2/?id=1--+



QUERY: SELECT * FROM TABLE_NAME WHERE ID= 1′–+ LIMIT 0,1

c) On lesson 3.

We need to first inject ‘) as discussed above to close the initial delimiter and then comment out rest of the query.

URL: http://localhost/Less-3/?id=1\--+



QUERY: SELECT * FROM TABLE_NAME WHERE ID=(‘ 1′) –+ ‘) LIMIT 0,1
