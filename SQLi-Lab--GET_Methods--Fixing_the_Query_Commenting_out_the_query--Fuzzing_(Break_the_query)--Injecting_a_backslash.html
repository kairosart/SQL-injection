Injecting a backslash



Backslash, “\” is an escape character to print characters which have special meaning in SQL..

Let's append “id=1\” to each lesson.

a) On lesson 1.

URL: http://localhost/Less-1/?id=1\

`

ERROR: "You have an error in your SQL syntax; check the manual that corresponds to your MySQL server version for the right syntax to use near ''1\' LIMIT 0,1' at line 1.

We observe that as our input was 1, a single quote is visible after that, indicating that single quotes are used as a wrapper for the strings.

QUERY: SELECT * FROM TABLE_NAME WHERE ID=’$ID’ LIMIT 0,1

b) On lesson 2.

URL: http://localhost/Less-2/?id=1\



ERROR: "You have an error in your SQL syntax; check the manual that corresponds to your MySQL server version for the right syntax to use near '\ LIMIT 0,1' at line 1".

After our input 1 there are no quotes, again proving that lesson 2 is an "integer-based injection” and does not need any quotes to break the query.

QUERY: SELECT * FROM TABLE_NAME WHERE ID=$ID LIMIT 0,1

c) On lesson 3.

URL: http://localhost/Less-3/?id=1\



ERROR: “You have an error in your SQL syntax; check the manual that corresponds to your MySQL server version for the right syntax to use near ''1\') LIMIT 0,1' at line 1”.

In this lesson, with the same input of 1, we see ‘) after our backslash, indicating that the developer of the application has wrapped the variable inside parentheses (‘ variable’).

QUERY: SELECT * FROM TABLE_NAME WHERE ID=(‘$ID’) LIMIT 0,1

d) On lesson 4.

URL: http://localhost/Less-4/?id=1\



You have an error in your SQL syntax; check the manual that corresponds to your MySQL server version for the right syntax to use near '"1\") LIMIT 0,1' at line 1

We see “) after our backslash indicating that the developer of the application has wrapped variable inside parentheses (“variable”).

QUERY: SELECT * FROM TABLE_NAME WHERE ID=(“$ID”) LIMIT 0,1
