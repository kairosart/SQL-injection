Error based: Single and double quote



Single quote

a) On lesson 1.

URL: http://localhost/Less-1/?id=1'



ERROR: "You have an error in your SQL syntax; check the manual that corresponds to your MySQL server version for the right syntax to use near ”1” LIMIT 0,1′ at line 1".

b) On lesson 2.

URL: http://localhost/Less-2/?id=1'



ERROR: "You have an error in your SQL syntax; check the manual that corresponds to your MySQL server version for the right syntax to use near ”1” LIMIT 0,1′ at line 1".

c) Lesson 3.

URL: http://localhost/Less-3/?id=1'



ERROR: "You have an error in your SQL syntax; check the manual that corresponds to your MySQL server version for the right syntax to use near ”1” LIMIT 0,1′ at line 1".

d) Lesson 4.

URL: http://localhost/Less-4/?id=1'



ERROR: It doesn't produce any alert with the injection of a single quote.


Double quote

If we do the same with double quote only Lesson 4 crashes the application and Less-1, Less-2, and Less-3 do not crash it anymore.


Conclusion

Less-1, Less-2, and Less-3 are reactive to a single-quote injection and Less-4 is to a double-quote injection.
