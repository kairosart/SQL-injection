SQL Queries vs Data



Let's suposse we have the following query:


http://example.com/color?colorid=4

we can divide the query in two sides.




When you execute this query you're sending the following code to the database:

select * from color where colorid=4


The number 4 is an untrusted data. The ‘craft’ of this. And here is the really risk of SQL Injection.
Instead of ths number you can add any other instruction or data.

Because the way it was constructed, the user can supply crafted input trying to make the original SQL statement execute further actions of the user’s choice.
