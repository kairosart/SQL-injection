Executing a SQL Injection Attack



We're going to explain in this section the components and steps of executing an SQL Injection Attack.

1) We have an attacker tha wants to get access to a system accoss HTTP protocol. This attacker send a request to a Web Server.
2) The server receives the request and construct a query that is where the attack is constructed.
3) Very ofrten there is a firewall between the Web Server and the Database. But as the query goes through the firewall all the way to the Database, this is one of the thing the attack must be successful.



4) The Database response back to the Web Server through the firewall and eventually back to the attacker.



This is the way an user comunicates with a Database in a common way.

5) When the Web Server talks to the Database must be Authenticated. The database has the user's credentials to acces to the Web. But if the credentials sent by the attacker are enough crafty, the Database can be compromised and the acces to the system might be a disaster.



6) The database and the technology use on the Web Sever can very really different. You can use ASP, PHP, JAVA, etc. on the Web Server and MYSQL, ORACLE, etc. or any other Database. That's why you have to investigate in the first step these aspects. The injection will be different in any case.
