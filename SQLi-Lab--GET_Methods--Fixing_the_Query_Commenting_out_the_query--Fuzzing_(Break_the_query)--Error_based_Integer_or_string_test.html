Error based: Integer or string test



Because the input parameter seems to be integer value, let us try to input a string value for ID parameter and observe the behavior.

a) On lesson 1.

URL: http://localhost/Less-1/?id=abc



Return: Empty set.

b) On lesson 2.

URL: http://localhost/Less-2/?id=abc



Return: it displays a MySQL error message on screen. “Unknown column 'abc' in 'where clause'”. It'is producing a MySQL alert, meaning that there are no quotes used around the input parameter, therefore integer values work fine in a query but strings cause an error.

c) On lesson 3.

URL: http://localhost/Less-3/?id=abc



Return: Empty set.

d) On lesson 4.

URL: http://localhost/Less-4/?id=abc



Return: Empty set.

We can assume that Less-1, -3, and -4 are using some sort of quotes to wrap around the user input. They consider the string entered as a non-existent value in the database and therefore respond with the empty set.

Deduction

• Lesson 1, Lesson 3, and Lesson 4 are “string-based injections”.
• Lesson 2 is an “integer-based injection”
